# InkStage 后端项目介绍

## 项目概述

InkStage 后端是一个基于 Spring Boot 4.0.2 构建的现代化内容管理系统后端，为前端应用提供完整的 API 支持。该后端系统实现了用户管理、文章管理、评论互动、通知系统、文件存储等核心功能，采用分层架构设计，具有良好的可扩展性和可维护性。

## 网站地址

https://www.inkstage.cn

目前仅支持用户名/密码注册登录，暂未开放手机号/邮箱注册登录。

## 架构设计

### 1. 分层架构

- **表示层**：Controller 层，处理 HTTP 请求和响应
- **业务逻辑层**：Service 层，实现核心业务逻辑
- **数据访问层**：Mapper 层，负责数据库操作
- **模型层**：Entity、DTO、VO 等数据模型

### 2. 核心模块

| 模块 | 主要职责 | 关键类 |
|------|---------|--------|
| 用户模块 | 用户注册、登录、个人资料管理 | UserController, UserService |
| 文章模块 | 文章创建、编辑、查询、管理 | ArticleController, ArticleService |
| 评论模块 | 评论创建、回复、管理 | CommentController, CommentService |
| 互动模块 | 点赞、收藏、关注 | SocialController, FollowService |
| 通知模块 | 系统通知、互动通知 | NotificationController, NotificationService |
| 搜索模块 | 文章搜索、热门搜索 | SearchController, SearchService |
| 推荐模块 | 文章推荐、个性化推荐 | RecommendController, PersonalRecommendService |
| 后台管理 | 用户、文章、评论等管理 | AdminUserController, AdminArticleController |

### 3. 数据流

```
HTTP 请求 → Controller → Service → Mapper → 数据库
                                     ↓
                                Redis 缓存
                                     ↓
                            RabbitMQ 消息队列
```

## 核心功能

### 1. 用户系统

- **用户注册**：支持邮箱注册，包含验证码验证
- **用户登录**：支持账号密码登录，返回 JWT 令牌
- **个人资料管理**：修改头像、昵称、签名等信息
- **用户认证**：基于 OAuth2 + JWT 的认证机制
- **权限管理**：基于角色的权限控制

### 2. 文章系统

- **文章创建**：支持 Markdown 格式，包含封面图上传
- **文章编辑**：支持修改文章内容、分类、标签
- **文章查询**：支持分页查询、按分类/标签查询
- **文章详情**：获取文章完整信息，包括作者、评论等
- **文章统计**：阅读量、点赞数、评论数统计

### 3. 互动系统

- **评论功能**：支持对文章发表评论，评论回复
- **点赞功能**：支持对文章和评论点赞
- **收藏功能**：支持收藏文章到文件夹
- **关注功能**：支持关注其他用户

### 4. 通知系统

- **系统通知**：系统公告、重要信息
- **互动通知**：点赞、评论、关注等通知
- **通知设置**：用户可自定义通知偏好
- **实时通知**：基于 WebSocket 的实时消息推送

### 5. 搜索系统

- **文章搜索**：支持关键词搜索文章
- **热门搜索**：展示热门搜索词
- **搜索历史**：记录用户搜索历史

### 6. 推荐系统

- **个性化推荐**：基于用户行为的个性化文章推荐
- **热门推荐**：基于热度的文章推荐
- **相关推荐**：基于文章标签的相关推荐

### 7. 文件系统

- **文件上传**：支持图片上传到 MinIO
- **文件管理**：管理用户上传的文件
- **图片处理**：支持图片预览、压缩等

### 8. 后台管理

- **用户管理**：查看、编辑、禁用用户
- **文章管理**：审核、编辑、删除文章
- **评论管理**：审核、删除评论
- **分类管理**：创建、编辑、删除分类
- **标签管理**：创建、编辑、删除标签
- **系统设置**：配置系统参数

## 部署方案

### 1. 本地开发

```bash
# 编译项目
mvn clean package

# 运行项目
java -jar target/inkStage-0.0.1-SNAPSHOT.jar
```

### 2. Docker 部署

**Dockerfile**：

```dockerfile
FROM openjdk:21-jdk-slim
VOLUME /tmp
COPY target/inkStage-0.0.1-SNAPSHOT.jar app.jar
ENTRYPOINT ["java","-Djava.security.egd=file:/dev/./urandom","-jar","/app.jar"]
```

**docker-compose.yml**：

```yaml
version: '3'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: inkstage
    ports:
      - "3306:3306"

  redis:
    image: redis:6.0
    ports:
      - "6379:6379"

  rabbitmq:
    image: rabbitmq:3.8-management
    ports:
      - "5672:5672"
      - "15672:15672"

  minio:
    image: minio/minio
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadmin
    command: server /data
    ports:
      - "9000:9000"
      - "9001:9001"

  backend:
    build: .
    ports:
      - "8080:8080"
    depends_on:
      - mysql
      - redis
      - rabbitmq
      - minio
```

### 3. 生产环境部署

- **服务器选择**：推荐使用云服务器，如阿里云、腾讯云
- **数据库**：使用云数据库 RDS，提高可靠性
- **缓存**：使用云 Redis 服务
- **消息队列**：使用云消息队列服务
- **对象存储**：使用云对象存储服务，如 OSS、COS 等
- **负载均衡**：配置负载均衡，提高系统可用性
- **SSL 证书**：配置 HTTPS，提高安全性

## API 设计

### 1. 接口规范

- **基础路径**：`/api`
- **版本控制**：通过路径参数实现，如 `/api/v1`
- **请求方法**：遵循 RESTful 规范
- **响应格式**：统一 JSON 格式

### 2. 响应结构

```json
{
  "code": 200,
  "message": "成功",
  "data": {}
}
```

### 3. 主要接口

| 模块 | 路径 | 方法 | 功能 |
|------|------|------|------|
| 用户 | `/api/front/auth/login` | POST | 用户登录 |
| 用户 | `/api/front/auth/register` | POST | 用户注册 |
| 文章 | `/api/front/articles` | GET | 获取文章列表 |
| 文章 | `/api/front/articles/{id}` | GET | 获取文章详情 |
| 文章 | `/api/front/articles` | POST | 创建文章 |
| 文章 | `/api/front/articles/{id}` | PUT | 更新文章 |
| 评论 | `/api/front/comments` | POST | 发表评论 |
| 评论 | `/api/front/articles/{articleId}/comments` | GET | 获取文章评论 |
| 互动 | `/api/front/articles/{id}/like` | POST | 点赞文章 |
| 互动 | `/api/front/articles/{id}/collect` | POST | 收藏文章 |
| 通知 | `/api/front/notifications` | GET | 获取通知列表 |
| 搜索 | `/api/front/search` | GET | 搜索文章 |
| 推荐 | `/api/front/recommend` | GET | 获取推荐文章 |
